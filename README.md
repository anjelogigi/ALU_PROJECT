# Parameterized ALU Project 

## Project Title

**Design and Verification of a Parameterized ALU**

## Project Overview

This project focuses on the design and verification of a **Parameterized Arithmetic Logic Unit (ALU)** using Verilog/SystemVerilog. The ALU is designed with configurable data width, making it reusable for different bit-width requirements such as 4-bit, 8-bit, or N-bit operations.

The ALU performs arithmetic, logical, comparison, shift, rotate, signed arithmetic, and multiplication-based operations. The design was verified using a simulation-based verification environment with a reference model, monitor, scoreboard, and comparator.

## Objective

The main objective of this project is to design a reusable ALU and verify its functionality across different input conditions and command operations.

The project also aims to check the correctness of the ALU output by comparing the DUT output with the expected output generated by the reference model.

The ALU supports the following major features:

## ALU Inputs and Outputs

### Inputs

- `OPA` : Operand A
- `OPB` : Operand B
- `CLK` : Clock signal
- `RST` : Reset signal
- `CE` : Clock enable
- `MODE` : Selects arithmetic or logical mode
- `CIN` : Carry input
- `CMD` : Command input for operation selection
- `INP_VALID` : Input validity signal

### Outputs

- `RES` : ALU result
- `COUT` : Carry output
- `OFLOW` : Overflow flag
- `G` : Greater-than flag
- `E` : Equal flag
- `L` : Less-than flag
- `ERR` : Error flag

## Working of the Design

The ALU works based on the command and mode inputs.

When reset is active, all output signals are cleared to their default values. When clock enable is active, the ALU checks the selected mode and command. Based on the command, the ALU performs the corresponding arithmetic or logical operation.

For arithmetic operations, the ALU performs addition, subtraction, increment, decrement, comparison, signed operations, and multiplication-based operations.

For logical operations, the ALU performs bitwise AND, NAND, OR, NOR, XOR, XNOR, NOT, shift, and rotate operations.

The `INP_VALID` signal is used to check whether the required operands are valid for the selected operation. If the input condition is not satisfied, the ALU sets the error flag.

## Verification Environment

The ALU was verified using a structured testbench environment.

The verification environment includes:

- **DUT**: The actual ALU design under verification.
- **Driver**: Applies input stimulus to the DUT.
- **Monitor**: Captures DUT outputs.
- **Reference Model**: Generates expected output based on the same input.
- **Comparator**: Compares DUT output and reference model output.
- **Scoreboard**: Displays PASS or FAIL based on comparison.

## Testbench Architecture

The testbench applies different input combinations to verify all ALU operations. The same inputs are given to both the DUT and the reference model.

The DUT output is compared with the reference output. If both outputs match, the test case passes. If there is a mismatch, the test case fails and the difference is displayed.
